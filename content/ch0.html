Special Topic: Java Command Line

Topic goals

  1. learn about basic Java commands for non-modular programmes
  2. learn about Java commands for modular programmes
  3. learn to discover & describe modules via the command line
  4. learn to use jdeps to list packages

1. learn about basic Java commands for non-modular programmes

checking Java version in the terminal

  • javac -version
  • java -version

Compile .java files

(if the compiled classes are to be placed in the same directory as the .java files)

  • javac example1.java
    (given that example1.java uses the default package, i.e. no package name is specified in it, and that we are in the same directory as the file)
  • javac package1/example2.java
    (given that example2.java declares that it is inside the package "package1" in it and that we are in the parent directory of the package1 directory)
  • javac package2/*.java
    (here the wildcard symbol * means all files that have the .java extension under "package2" are to be compiled)

(if the compiled classes are to be placed in a specified class path)

  • javac -d classDir package3/example3.java

(if compilation involves dependencies (i.e. external libraries, can be directories or jars))

  • javac -cp dependencyDir -d classDir example4.java
  • javac -cp dependencyJar -d classDir example4.java

Run a compiled .class file

(if the classes are placed in the same directory as the files)

  • java package1/example2.java
